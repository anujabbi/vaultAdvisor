# VaultAdvisor — Design Specification

**Date:** 2026-06-10
**Status:** Approved (brainstormed interactively; final scope decisions confirmed by owner)
**Owner:** anujabbi

## 1. Product definition

VaultAdvisor is a desktop personal-finance advisor for US users. The core pitch:

> Your financial documents never leave your machine. There is no VaultAdvisor server, no
> account, no cloud database. The only party that ever sees your data is the AI provider
> you already trust — Anthropic — via your own sign-in. We give you real, personalized
> investing and tax advice, locally.

**Audience:** the owner, family, and friends (informal multi-user via separate OS users /
app data dirs), distributed publicly as an open-source app with a marketing page.

**Jurisdiction:** United States tax and investment rules only (v1).

**Trust model:** the app is a local front-end. Data is stored in a local SQLite database in
the user's app-data folder. LLM calls go directly from the user's machine to Anthropic,
authenticated by the user's own Claude subscription ("Sign in with Claude" via the Claude
Agent SDK, drawing from the user's Agent SDK credit pool). No proxy server, no telemetry.

**Compliance posture:** all output is framed as educational analysis, not professional
financial, tax, or legal advice. A persistent disclaimer appears in the app footer and on
each advice card's expanded view.

## 2. Core functions (v1)

1. **Hero scenarios (the lure).** First-launch experience shows sample-data cards built
   from a fictional persona: "Saved $4,100 via tax-loss harvesting", "30% overweight tech
   without knowing it", "Idle cash costing $2,300/yr". Each card is interactive (expands to
   show the full sample analysis) and ends with an invitation: *upload your statement and
   see yours*. Clicking the invitation routes to document intake scoped to that scenario.

2. **Document intake.** Drag-and-drop or file-picker upload of PDFs and CSVs:
   - Brokerage / 401(k) statements → holdings, lots, cost basis, fees
   - Tax return (1040) → AGI, bracket, deductions, filing status
   - Pay stubs / W-2 → income, withholding, 401(k) contribution pacing
   - Bank statements → cash balances, interest rates
   The Claude Agent SDK parses documents into structured JSON (schema-constrained).
   The user reviews and corrects the extraction in a review screen before anything is
   saved to the local database. Original files are copied into a local vault folder.

3. **Conversational profiling.** After an upload is confirmed, the advisor opens a short,
   data-grounded conversation ("You're heavy in AI names — thematic on purpose? Other
   themes you believe in?") to capture goals, horizon, risk appetite, and beliefs/
   constraints. No upfront survey. The resulting profile is stored as structured facts,
   visible and editable on a Profile page. Every recommendation cites which profile facts
   it relied on.

4. **Advice dashboard.** Home screen once data exists:
   - Top: net worth, asset allocation donut, tax picture summary.
   - Below: a feed of recommendation cards generated by the advice engine, e.g.
     rebalancing, tax-loss harvesting, contribution-limit gaps, idle-cash drag,
     withholding checkup, Roth-vs-traditional. Cards "light up" as the data and profile
     needed for them becomes available; locked cards show what upload would unlock them.
   - Each card expands into: **the why** (analysis + math), **citations** (links to
     reputable sources — IRS.gov, FINRA, Bogleheads, Fidelity/Schwab/Vanguard education
     pages — gathered via the Agent SDK's web search), and an **execution checklist**:
     concrete manual steps ("Sell lot #3 of XYZ ☐ → Confirm settlement ☐ → Mark done")
     the user checks off in-app. Completion state is tracked; the advisor follows up on
     incomplete checklists and recomputes affected advice when one is marked done.

5. **Advisor chat.** Secondary chat pane (slide-over) with full local-data context for
   deep dives ("what if I sell X this year vs next?"). Can also be opened scoped to a
   specific card.

**Out of scope (v1):** account linking/Plaid, trade execution, budgeting/expense
categorization, multi-jurisdiction tax, mobile, OpenAI sign-in (see §4).

## 3. Architecture

**Stack:** Electron (Windows + Mac), React 18 + TypeScript + Vite renderer, Node main
process, SQLite via better-sqlite3 in `app.getPath('userData')/vaultadvisor.db`,
electron-builder for packaging.

**Process split:**
- **Renderer (React):** all UI. No direct network or filesystem access; talks to main via
  a typed IPC bridge (contextBridge preload, `window.vault.*`).
- **Main (Node):** SQLite store, document vault folder, and the **advice engine** — a
  service wrapping the Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`). All LLM
  calls, document parsing, web-search citation gathering, and advice generation happen
  here. Streaming results forwarded to renderer over IPC channels.

**Modules (main process):**
- `auth/` — Sign in with Claude flow (Agent SDK OAuth login), session/token state in OS
  keychain (keytar/safeStorage). Provider interface `LlmProvider` so OpenAI can be added.
- `store/` — SQLite schema + repositories: documents, holdings, lots, accounts, income,
  tax_facts, cash, profile_facts, advice_cards, checklist_items, chat_threads.
- `ingest/` — upload handling, file vault, LLM extraction with JSON-schema-constrained
  output, review/confirm workflow.
- `advisor/` — advice generation. For each advice domain a prompt + Claude finance/Agent
  Skills produce: finding, math, citations (via web search tool), checklist steps.
  Deterministic pre-computations (allocation %, bracket math, contribution-limit gaps)
  are done in TypeScript and fed to the LLM so numbers are exact; the LLM explains,
  personalizes against profile facts, and cites.
- `chat/` — advisor chat sessions with local-context assembly.

**Advice card lifecycle:** `locked → available → generated → acted-on/dismissed/stale`.
Cards regenerate when underlying data changes (new upload, checklist completion, profile
edit).

**Data flow (upload):** file → vault copy → Agent SDK parse (streamed) → review screen →
user confirms/corrects → SQLite → profiling conversation → advice engine re-evaluates
card availability → dashboard updates.

## 4. LLM provider & auth

- **Primary:** Claude, via the Claude Agent SDK with "Sign in with Claude" (the user's
  Claude subscription; usage draws from the Agent SDK credit pool effective 2026-06-15).
  Default model: the SDK session default (Opus-tier); web search enabled for citations.
- **OpenAI:** Sign in with ChatGPT is not generally available to third-party apps as of
  2026-06-10 (Codex CLI preview only). The UI shows OpenAI as "coming soon"; the
  `LlmProvider` interface is the seam where it lands. **No API-key entry ships in v1**
  (owner decision: sign-in only).
- Auth state, document vault, and DB are all per-OS-user; "family use" = each person uses
  their own machine/OS account and their own Claude subscription.

## 5. Error handling

- LLM/network errors: cards show retriable error state; chat shows inline error; parsing
  failures fall back to "couldn't read this document — try a CSV export" guidance.
- Extraction confidence: low-confidence fields highlighted in the review screen.
- Auth expiry: re-auth prompt; app remains usable read-only (data is local).
- Agent SDK credit exhaustion: surfaced with a plain explanation and a link to the user's
  Claude plan page.

## 6. Testing

- Unit tests (Vitest) for deterministic finance math (allocation, bracket, limits,
  loss-harvest detection) and store repositories.
- Fixture-based tests for extraction schemas (sample statement JSON → DB rows).
- Manual smoke checklist for the Electron shell (launch, upload, sign-in) per release.

## 7. Marketing page & distribution

- `site/` — static, distinctive landing page (privacy-first pitch, hero scenario
  screenshots, "Download for Windows / Mac" buttons) deployed to GitHub Pages via a
  workflow on pushes to `main`.
- Releases: electron-builder artifacts attached to GitHub Releases; download buttons link
  to the latest release.
- Repo: public, `anujabbi/vaultAdvisor`. Open source reinforces the trust pitch.
